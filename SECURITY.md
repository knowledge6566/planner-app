# Security Policy

## Supported Versions

The following versions of this project are currently supported with security updates:

| Version | Supported |
|---------|-----------|
| Latest | ✅ Yes |
| Older Versions | ❌ No |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

### How to Report

- Open a private security report through GitHub Security Advisories (if enabled), or
- Email: shauryasingh2029@gmail.com

Please include:
- A detailed description of the vulnerability.
- Steps to reproduce the issue.
- Any proof-of-concept or screenshots (if applicable).
- The potential impact of the vulnerability.

## Response Timeline

- Initial acknowledgment: Within 48 hours
- Status update: Within 5 business days
- Fix (if confirmed): As soon as reasonably possible

## Responsible Disclosure

Please do not publicly disclose security vulnerabilities until they have been investigated and resolved. Responsible disclosure helps protect all users of this project.

## Scope

This policy applies to:
- Source code
- Dependencies
- Authentication
- API endpoints
- Configuration files

Third-party services are governed by their own security policies.

## Security Best Practices

- Keep dependencies up to date.
- Never commit API keys or secrets.
- Use HTTPS in production.
- Enable environment variables for sensitive credentials.
- Regularly review access permissions.

Thank you for helping keep this project secure.
